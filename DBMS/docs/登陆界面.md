#### 登陆界面

###### 用户信息表的结构

用户信息表的结构如下

| UserName  | string |
| --------- | ------ |
| Password  | string |
| Level     | int    |
| Conformed | int    |

其中用户名、密码即为注册的用户名密码；level属性为用户等级，0代表管理员级、1代表普通用户级；Conformed属性代表该条是否有效。

###### 注册流程

用户通过注册页面进行注册，注册时提供用户名、密码及注册的用户等级从而生成一条注册申请（用户名不接受重名）。

管理员可以在登陆后查看用户的注册申请，可以通过指定用户名将其记录变为有效或删除某用户。
